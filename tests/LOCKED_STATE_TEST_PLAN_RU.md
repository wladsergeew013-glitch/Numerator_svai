# LOCKED STATE TEST PLAN — v54

Цель: зафиксировать рабочее состояние после v53 и перед следующими доработками ловить потерю готового функционала автоматически.

## Быстрая проверка перед каждой правкой

```bat
tools\02_check_frontend_contract.bat
tools\07_check_locked_state.bat
```

## Полная проверка перед отдачей патча

```bat
tools\07_check_locked_state.bat --build
```

## Что защищает `02_check_frontend_contract`

- верхнюю ленту, вкладки, команды и resize ленты;
- диспетчер групп, пайплайны, кнопки действий и методы нумерации;
- dock/pinned окна и z-index поверх закреплённых панелей;
- импорт/экспорт CSV/Excel/nanoCAD, скан nanoCAD для импорта и экспорта;
- проверку модели и режим подсветки пустых точек;
- ручные связи, маршрут/вектор, старт/финиш, сохранение routePointIds;
- авто/ручной контур группы, `Shift`/snap-фантом и world-based CAD-контур.

## Что защищает `07_check_locked_state`

- структуру проекта и наличие корневых контрактов `RULES.md` / `FUNCTIONALITY_LOCK_RU.md`;
- компиляцию всех backend Python-модулей;
- CSV-импорт, разделение `id` и номера сваи, парсинг десятичных запятых;
- нумерацию рядов, глобальную последовательность между группами и сохранение ручных номеров;
- route/vector backend-алгоритмы на минимальных примерах;
- соответствие frontend API-клиента backend-эндпоинтам;
- наличие nanoCAD-блоков, Model Studio, базовой точки, допуска 1 мм и export-групп;
- наличие зависимостей `xlsx`, `konva`, `zustand`, `vite`, `typescript`;
- наличие всех numbered tools `01..07`.

## v59 — ручной вектор и фантомы связей

1. В группе выбрать метод **Вектор**, нажать **Нарисовать**, зажать ЛКМ и провести линию по полю.
2. Убедиться, что видна оранжевая draft-линия, `Enter` применяет путь, а точки нумеруются вдоль нарисованной траектории.
3. `Esc` в режиме рисования вектора должен отменять draft без изменения группы.
4. При выборе ручного старта/финиша клик по области стрелок предпросмотра не должен выбирать связь — выбираются только точки.
5. После назначения ручной связи не должно оставаться фантомной второй связи/поля; остаётся только реальная ручная связь в preview.

## v60 — ярлык EXE, проверка 07 и редактируемый вектор

1. `tools\07_check_locked_state.bat` должен запускаться без ошибки вида `\"...python.exe\" is not recognized`; Python вызывается через `PY_EXE/PY_LAUNCH` и `call :run_python`.
2. После сборки EXE запустить `tools\08_create_desktop_shortcut.bat`; на рабочем столе должен появиться ярлык `Pile Numbering App` с иконкой из `tools\app_icon.ico`.
3. В группе с методом **Вектор** после `Нарисовать → Enter` оранжевая линия должна остаться на рабочем поле.
4. У сохранённой векторной линии должны быть характерные точки `V1/Vn`; их можно перетаскивать мышью без перерисовки всего вектора.
5. После перетаскивания характерной точки маршрут/preview пересчитываются, а путь остаётся в `group.numbering.vectorPath`.
6. В режимах выбора старт/финиш/ручная связь векторные handle-точки не должны мешать выбору свай.

   - Настройки → `Автосохр.`;
   - выбрать папку или ввести путь;

## v62 — ручная проверка вектора и ярлыка

1. Собрать EXE через `tools\06_build_exe.bat`.
2. Запустить `tools\08_create_desktop_shortcut.bat` и проверить, что появился `dist\Pile Numbering App.lnk`.
3. Открыть группу с методом `Вектор`, нажать `Нарисовать`:
   - ЛКМ добавляет только вершины/сегменты, не кисточные сотни точек;
   - `Shift` делает сегмент ортогональным;
   - `Ctrl+Z` удаляет последнюю вершину;
   - `Enter` сохраняет путь.
4. Нажать `Редактировать`:
   - точки свай затемнены;
   - обычное выделение точек не работает;
   - крайние вершины имеют кнопку `+` для продолжения;
   - выбранный сегмент имеет кнопку `+` в середине для вставки вершины;
   - `Delete` / ПКМ удаляет выбранную вершину.

## v63 — ручная проверка

1. В группе с методом `Вектор` нажать `Редактировать`.
2. Кликнуть рядом с вершиной: выбирается вершина, а не свая.
