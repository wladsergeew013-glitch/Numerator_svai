# Функциональный контракт нумератора свайного поля

Дата фиксации: v36. Назначение файла — держать уже готовый функционал в одном месте и сверяться с ним перед каждой следующей правкой, чтобы не терять рабочие команды.

## 1. Верхняя лента

Есть два режима отображения ленты:

- **По вкладкам** — показывает команды только выбранной вкладки.
- **Все секции** — показывает все секции подряд.

Вкладки верхнего уровня:

- **Файл**.
- **Редактирование / группы / нумерация**.
- **Настройки**.
- **История**.

Секции можно сворачивать/разворачивать. При сворачивании секция должна реально сужаться, без вертикального переворота заголовка и без налезания кнопок.

## 2. Файл / проект

Команды:

- **Создать** — создать новый пустой проект.
- **Открыть** — открыть `.pilenum.json` / `.json` или импортировать `.csv` как новый проект.
- **Проекты** — открыть список локальных проектов из папки `projects`.
- **Импорт** — добавить CSV-точки в текущий проект, не обнуляя существующее поле.
- **Сохранить** — сохранить текущий проект в локальную папку `projects`.
- **Экспорт** — скачать текущий проект как `.pilenum.json`.

Текущий проект показывается в верхней компактной строке. Для нового несохранённого проекта показывается статус, что проект ещё не сохранён.

## 3. Редактирование точек

Команды:

- **Точка** — создать точку по координатам. Новая точка должна получать активную группу, если она есть и не заблокирована.
- **Удалить** — удалить выбранные точки. Клавиша `Delete` делает то же самое, кроме случая выбранной ручной связи нумерации.
- **Переместить** — переместить выбранные точки по оси X/Y на заданное расстояние.
- **Копировать** — копировать выбранные точки по смещению или через базовую точку и точку вставки. Базовая точка и точка вставки выбираются из существующих точек по ближайшей точке.
- **Коп.св-ва** — копировать свойства от исходной точки к одной или нескольким точкам-приёмникам. Сейчас копируется группа.
- **Инфо** — режим выбора ближайшей точки и просмотра/редактирования её свойств.

Для операций выбора точек:

- одиночный клик выбирает точку;
- рамка выделения работает по текущему режиму выделения;
- в режиме `Инфо` автоназначение группы рамкой не срабатывает.

## 4. Диспетчер групп

Группа имеет:

- имя;
- порядок;
- цвет;
- видимость/активность;
- блокировку;
- метод нумерации;
- старт и финиш;
- ручные связи;
- ручной или автоматический контур;
- ручные номера отдельных точек.

Команды диспетчера:

- **Создать группу** — добавить группу с первым свободным CAD-цветом.
- **Развернуть все / Свернуть все** — раскрыть или свернуть карточки групп.
- **Назначить** — команда верхней ленты. Если есть выделенные точки, один раз назначает их в активную группу. Если выделения нет, включает/выключает режим автоназначения рамкой. Режим не зависит от открытого диспетчера групп.
- **↑ / ↓** — изменить порядок группы.
- **Снять ручные** — снять ручные номера точек группы.
- **Удалить** — удалить группу, точки группы становятся без группы.
- **🔒 / 🔓** — заблокировать/разблокировать группу.

Если группа заблокирована:

- её точки нельзя удалять/двигать массовыми командами;
- нельзя менять ручные связи;
- нельзя менять старт/финиш;
- нельзя менять ручной контур;
- нельзя двигать выноски номеров;
- двойной клик по точке не должен открывать ручной ввод номера.

## 5. Методы нумерации группы

В интерфейсе группы доступны только 4 метода:

- **Ряды**.
- **Столбцы**.
- **Маршрут**.
- **Вектор**.

Метод **Ручной** не является методом группы. Ручная нумерация — это ручная фиксация номера отдельной точки поверх выбранного метода.

### Ряды

Показываются только настройки:

- допуск рядов по Y;
- направления рядов:
  - слева направо, сверху вниз;
  - справа налево, сверху вниз;
  - слева направо, снизу вверх;
  - справа налево, снизу вверх;
  - змейка от левого верхнего угла;
  - змейка от правого верхнего угла.

Не должны показываться настройки колонок, маршрута или вектора.

### Столбцы

Показываются только настройки:

- допуск колонок по X;
- направления колонок:
  - сверху вниз, слева направо;
  - снизу вверх, слева направо.

Не должны показываться допуск рядов, настройки маршрута или настройки вектора.

### Маршрут

Показываются только настройки:

- оптимизировать маршрут;
- старт группы;
- финиш группы;
- контур группы.

Допуски рядов/колонок и направления рядов/колонок в этом методе не показываются.

### Вектор

Показываются только настройки:

- допуск до векторной линии;
- количество точек в векторной линии;
- старт группы;
- финиш группы;
- контур группы.

Допуски рядов/колонок и направления рядов/колонок в этом методе не показываются.

## 6. Старт и финиш группы

Для каждой группы есть старт и финиш:

- **Авто** — выбирается автоматически;
- **Выбрать** — выбрать ближайшую существующую точку активной группы.

Если выбранная стартовая/финишная точка позже ушла в другую группу, ссылка должна сброситься в `Авто`.

Авто-старт следующей группы должен учитывать финиш предыдущей непустой группы, если старт следующей группы не задан вручную.

Метки старта/финиша:

- показываются выносками;
- их можно перетаскивать;
- подпись должна различать `авто` и `вручную`;
- смещение должно масштабироваться естественно при zoom in / zoom out.

## 7. Контур группы

Есть два режима контура:

- **Авто-контур** — строится автоматически по точкам группы.
- **Ручной контур** — пользователь сам кликает вершины контура.

Ручной контур:

- команда запускается из карточки группы;
- ЛКМ добавляет вершину;
- клик по первой вершине или `Enter` замыкает контур;
- `Esc` отменяет рисование;
- ПКМ и средняя кнопка не отменяют рисование и должны позволять панорамировать поле;
- выноска имени группы должна цепляться к первой вершине текущего контура.

## 8. Нумерация / предпросмотр / связи

Команды секции нумерации:

- **Весь путь** — показать всю траекторию активной группы.
- **Мультик** — запустить анимированный просмотр.
- **Мультик / Пауза** — если мультик идёт, повторное нажатие ставит паузу; если пауза стоит, повторное нажатие продолжает с текущего места, а не начинает сначала.
- **Очистить** — открыть режим удаления ручных связей.
- **Группа** — применить нумерацию к активной группе.
- **Все** — применить нумерацию ко всем непустым группам по порядку.

`Esc`:

- если выбрана/редактируется связь — сначала снимает выбор связи;
- если открыт режим очистки связей — закрывает режим очистки;
- если идёт предпросмотр — сбрасывает предпросмотр;
- если рисуется контур — отменяет рисование контура.

Ручные связи:

- создаются кликом по стрелке пути и выбором новой точки-приёмника;
- начальная точка связи фиксируется, меняется только конечная;
- нельзя привязать связь назад к точке, которая уже была раньше в маршруте от выбранного источника;
- ручные связи отличаются цветом от автоматических;
- выбранная связь подсвечивается отдельным цветом;
- выбранную ручную связь можно удалить `Delete / Backspace` без удаления точек.

Очистка ручных связей:

- окно очистки должно появляться в рабочей области, не внутри верхней панели;
- считаются только валидные уникальные ручные связи активной группы;
- можно выбрать, какие связи удалить;
- связи, которые удалятся, подсвечиваются;
- точки, не участвующие в ручных связях, затемняются.

## 9. Ручная нумерация точки и выносы номеров

Ручной номер точки:

- двойной клик по точке открывает CAD-форму ручного номера;
- можно задать ручной номер;
- можно снять ручной номер;
- точка с ручным номером помечается как ручная;
- пересчёт нумерации не должен затирать ручную фиксацию.

Вынос номера:

- если выделена одна точка, можно тянуть плашку её номера;
- при сдвиге появляется пунктирная линия от точки к номеру;
- вынос можно сбросить из CAD-формы точки или окна информации;
- вынос не должен некорректно масштабироваться при zoom.

## 10. Настройки рабочего поля

Настройки должны сохраняться не как временный источник истины в `localStorage`, а в проекте и/или `config/user_config.json`.

Группы настроек:

- фон;
- сетка;
- номера;
- метки;
- предпросмотр/стрелки;
- контур группы;
- выделение.

При наведении на настройки соответствующий объект на поле должен подсвечиваться, чтобы было понятно, что именно меняется.

## 11. Журнал и отмена

`Ctrl+Z` / команда **Отмена** должны возвращать предыдущее состояние, если операция положила снимок в history.

Команда **Повтор** возвращает отменённое действие.

Журнал должен показывать и позволять восстановить состояние после операций текущей сессии. Старые записи из файла могут быть только информационными, если для них нет снимка состояния.

В журнал должны попадать:

- открытие проекта;
- создание проекта;
- импорт CSV как нового проекта;
- добавление CSV в текущий проект;
- создание точки;
- удаление точек;
- перемещение точек;
- копирование точек;
- копирование свойств группы;
- создание группы;
- удаление группы;
- изменение порядка групп;
- назначение точек в группу;
- блокировка/разблокировка группы;
- ручная связь нумерации;
- удаление ручной связи;
- очистка выбранных ручных связей;
- ручной контур группы;
- сброс ручного контура;
- ручной номер точки;
- снятие ручного номера точки;
- снятие ручных номеров группы;
- сдвиг выноса номера;
- нумерация группы;
- нумерация всех групп;
- сохранение локального проекта.

## 12. Регрессионный чек-лист перед новой правкой

Перед каждой следующей сборкой нужно проверять минимум:

1. Верхняя лента не сломана, секции не налезают друг на друга.
2. В карточке группы у `Ряды` видны только настройки рядов.
3. В карточке группы у `Столбцы` видны только настройки столбцов.
4. У `Маршрут` нет допусков рядов/колонок и направлений рядов/колонок.
5. У `Вектор` нет допусков рядов/колонок и направлений рядов/колонок.
6. Старт/финиш можно выбрать и сбросить в авто.
7. Ручной контур рисуется, замыкается, отменяется через `Esc`, не блокирует панорамирование ПКМ/средней кнопкой.
8. Выноска группы цепляется к первой вершине контура.
9. Весь путь/мультик/пауза работают; пауза продолжает мультик с текущего места.
10. Ручную связь можно создать, выбрать, удалить `Delete`, и при этом точки не удаляются.
11. Очистка ручных связей открывает окно в рабочей области, правильно считает связи и затемняет лишние точки.
12. Заблокированная группа не даёт редактировать связи, контуры, номера и точки.
13. Журнал пополняется по ключевым операциям, а `Ctrl+Z` откатывает последние изменения.


## Регрессия: ручная связь + zoom/pan

- При активном режиме выбора конца ручной связи колесо мыши/zoom не должен создавать "мираж" дублированных точек.
- Wheel очищает только временный draft-жест и рамку, но не выключает сам режим связи.
- В режиме редактирования ручной связи preview-кружки маршрута поверх основных точек не рисуются; остаются линии маршрута и выбранная связь.

### Дополнение v33 — читаемость ribbon и диспетчера групп

- Подпись команды на верхней ленте должна быть отдельной строкой под плиткой иконки, а не внутри рамки самой кнопки.
- Активное состояние команды должно подсвечивать плитку иконки. Для команды **Назначить** активное состояние означает включённое автоназначение рамкой.
- В диспетчере групп не должно быть чекбокса **Назначать выделение сразу**. Это управление находится только на верхней ленте.
- Диспетчер групп визуально разделён на блоки: **Пайплайны**, **Действия**, **Группы пайплайна**.
- При увеличении ширины диспетчер должен использовать место: список групп и настройки раскладываются в несколько колонок, а не остаются узкой вертикальной простынёй.

### Дополнение v34 — диспетчер групп и читаемость ribbon

- В блоке **Действия** диспетчера групп есть команда **Назначить**, синхронизированная с одноимённой командой верхней ленты.
- Если есть выделенные точки, **Назначить** назначает выделение в активную группу один раз.
- Если выделения нет, **Назначить** включает/выключает автоназначение рамкой. Активное состояние должно совпадать в верхней ленте и в диспетчере.
- Список пайплайнов в диспетчере отображается выпадающим списком, а не рядом вкладок/плиток.
- В карточке группы внутри блока **Группы пайплайна** не показывается отдельное поле выбора пайплайна: активный пайплайн уже выбран сверху.
- Карточки групп в **Группах пайплайна** идут по одной на всю ширину списка. Свёрнутая карточка должна подтягиваться по высоте и не оставлять пустую ячейку от соседней раскрытой карточки.
- Команды верхней ленты имеют единую ширину, рассчитанную под длинные подписи вроде **Переместить**, и не должны разрывать слово на последнюю букву.
- Все секции верхней ленты используют единый принцип группировки команд с заголовком подгруппы, чтобы кнопки стояли на одной высоте и читались одинаково.


### Дополнение v35 — докинг диспетчера, dropdown пайплайнов и настройка ленты

- Закреплённый слева/справа диспетчер групп не должен сжиматься до ширины, при которой кнопки и поля налезают друг на друга. Минимальная ширина диспетчера при докинге должна сохранять читаемость блоков **Пайплайны**, **Действия**, **Группы пайплайна**.
- Выбор пайплайна в диспетчере делается тёмным выпадающим меню внутри интерфейса, без отдельного серого поля переименования активного пайплайна.
- Верхняя лента имеет настройку плотности кнопок: **Плотно**, **Нормально**, **Свободно**. Плотность управляет высотой ячеек и расстоянием между командами, а ручное изменение высоты ленты масштабирует команды без обрезания подписей.
- Если все секции текущей вкладки ленты свёрнуты, в рабочей области ленты должен появляться явный блок восстановления **Развернуть вкладку**, чтобы пользователь не терял интерфейс.
- Команда **Сброс ленты** возвращает вид ленты к безопасному читаемому состоянию: режим по вкладкам, секции раскрыты, плотность компактная, высота стандартная.

### Дополнение v36 — масштабирование ленты, подтверждение удаления и вкладка иконок

- При изменении высоты верхней ленты кнопки, плитки и подписи должны пропорционально уменьшаться/увеличиваться по высоте, а не оставаться прежними до момента сжатия по ширине.
- Активное состояние команды подсвечивает плитку иконки, но не раздувает внешний контейнер команды.
- Клик по зажимной команде не должен сбрасывать горизонтальный scroll ленты в начало.
- Окно подтверждения удаления группы должно появляться поверх рабочей области; кнопка удаления в диспетчере не должна выглядеть как неработающая.
- Вкладка настроек **Иконки** должна оставаться табличной: иконка, название команды, id команды, выбор файла, загрузка/сброс.

### Дополнение v37 — якорение ribbon по высоте и кнопки действий диспетчера

- При изменении высоты верхней ленты секции, подгруппы, контейнеры кнопок, плитки и подписи должны сжиматься/растягиваться вместе, без отставания соседних секций и без налезания на рабочее поле.
- Свёрнутая секция ленты должна оставлять читаемое горизонтальное название секции и кнопку разворота; название не должно пропадать и не должно поворачиваться вертикально.
- Кнопки блока **Действия** в диспетчере групп стилизованы как CAD-команды с иконкой и текстом, но логика команд остаётся прежней: создать группу, назначить/автоназначить, авторазбить, развернуть все, свернуть все.

## Дополнение v51 — авто-контур, фантомная snap-точка и общий скан nanoCAD

- Автоматический контур группы больше не должен строиться как широкие полосы на весь ряд/колонку в мировых координатах. Контур строится в экранных координатах как union прямоугольных коридоров вокруг соседних свай, чтобы визуальный отступ не превращался в огромные миллиметры на малом zoom.
- Авто-контур обязан оставаться ортогональным CAD-многоугольником, но не должен накрывать большие пустые области между ветками группы и не должен терять отдельные ветки L/U-образного поля.
- При ручном рисовании контура должна показываться фантомная точка: пользователь видит, куда реально будет поставлена вершина до клика. В режиме `Shift` фантомная точка показывает ortho-привязку к ближайшей точке/вершине или свободному положению.

- Списки типов блоков/объектов и атрибутов/параметров, полученные сканом, общие для импорта и экспорта; экспорт не должен требовать ручного ввода имени блока, если скан уже выполнен.
- Smoke-check должен ловить потерю экранного авто-контура, фантомной точки ручного контура и export-скана nanoCAD.

## Дополнение v52 — быстрый ортогональный авто-контур и Shift-проекция ручной вершины

- Автоматический контур активной группы не должен строиться по прямоугольнику каждой сваи: это даёт лаги, зубцы и потерю контура при zoom. Базовый алгоритм — укрупнённые горизонтальные/вертикальные CAD-полосы по рядам и колонкам, плюс короткие ортогональные перемычки между компонентами.
- После построения авто-контура нельзя применять общий Douglas-Peucker simplifier, потому что он превращает ортогональные углы в диагонали и может визуально «съедать» контур при приближении. Допускается только ортогональное удаление дублей и коллинеарных точек.
- Ручной контур в режиме `Shift` работает как проекция: курсор выбирает ближайшую точку/вершину, а фактическая фантомная вершина ставится на пересечении этой точки с горизонтальной/вертикальной осью от предыдущей вершины.
- Фантомная точка ручного контура должна показывать именно итоговую координату, которая будет добавлена по клику. Линия от найденной точки/вершины до фантома показывает пользователю, откуда взята привязка.
- Smoke-check должен ловить потерю укрупнённых CAD-полос, запрет диагонального simplify для авто-контура и Shift-проекцию через найденную точку/вершину.

## Дополнение v53 — стабильный world-based CAD-контур без зубцов при zoom

- Автоматический контур активной группы больше не должен пересчитывать форму в экранных пикселях по каждой свае: это меняет геометрию при приближении/отдалении и снова даёт лаги.
- Базовый авто-контур строится в мировых координатах как небольшое число крупных CAD-прямоугольников: горизонтальные полки, вертикальные стойки и при необходимости ортогональные перемычки между компонентами.
- Отступ авто-контура вычисляется от характерного шага свайного поля, а настройка padding работает как масштаб этого отступа. Поэтому контур остаётся на одном месте при zoom/pan.
- Алгоритм обязан сначала пытаться покрыть группу крупными горизонтальными/вертикальными полосами, затем расширять ближайшую полосу под одиночные выступающие точки, а не создавать зубец вокруг каждой отдельной сваи.
- Ручной контур остаётся приоритетным: если в `group.meta.manualOutline` есть 3+ вершины, авто-контур не строится.
- Smoke-check должен ловить потерю `makeGroupOutlineWorldCadBands`, оценки шага поля `estimateNearestSpacingWorld` и объединения крупных прямоугольников через `connectOutlineRects`.

- В импорте Model Studio пользователь выбирает группы объектов чекбоксами; для каждой выбранной группы можно задать свой параметр чтения номера.
- В экспорте Model Studio выбранные группы объектов обновляются одним общим параметром записи номера.
- Запись параметров Model Studio должна идти по документированному COM-пути `Object.Element.Parameters.SetParameter(name, value[, caption, comment])`; после записи нужно пробовать `UpdateGraphics`. Прямая запись `Parameter.Value` допускается только как fallback.
- При повторном импорте из Model Studio приложение сопоставляет новые точки с текущим проектом по координатам: совпавшие точки можно заменить или оставить, новые объекты добавляются.
- При закреплении dock-окон ширина, зарезервированная рабочим полем, должна совпадать с фактической шириной закреплённых окон. Нельзя масштабировать dock-резерв отдельно от видимых панелей, иначе панели перекрывают нижний левый индикатор осей/сетку.


### Дополнение v79 — восстановление контракта Toolbar после Model Studio UI-патча

- Патчи Model Studio не должны удалять функции ленты/экспорта: `downloadProjectJson`, `showSaveFilePicker`, `saveProjectJson`, `openImportDialog`, `openExportDialog`, `confirmImportFromFile`, `refreshNanoCadBlocks`, `pickNanoCadSample`, `RibbonSection`, `toolbarStyle`.
- Экспорт JSON в EXE остаётся по цепочке: `window.pywebview.api.saveProjectJson` -> `showSaveFilePicker` -> fallback через download-link.
- UI-правки Model Studio считаются валидными только вместе с сохранением общего контракта верхней ленты и import/export workflow.

### Дополнение v80 — dock viewport и индикатор осей

- Рабочее поле при закреплении окон считается как отдельный fixed-viewport между ribbon и строкой состояния.
- Закрепленные окна слева/справа сужают именно видимую область Canvas, а не накладываются поверх нее.
- Индикатор осей X/Y в левом нижнем углу выводится поверх Canvas как DOM-overlay внутри видимой области рабочего поля, поэтому он не должен пропадать под закрепленным окном.
- При закрытии/откреплении окна dock-reserve пересчитывается только по реально открытым dock-панелям.

### Дополнение v81 — dock reserve по фактическим DOM-панелям

- Рабочее поле не полагается только на временный dock-registry в localStorage.
- При открытии, закрытии, закреплении и изменении размера окон резерв рабочего поля пересчитывается по фактическим `.docked-panel.docked-left/.docked-right` в DOM.
- Если dock-registry ещё не успел обновиться, но панель уже видима, Canvas всё равно сдвигается за реальную границу панели.
- Дополнительный DOM-индикатор осей из v80 убран: используется один штатный Konva-индикатор X/Y внутри Canvas, без дублей.
